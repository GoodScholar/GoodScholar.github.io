## SQLAlchemy 基础

```python
from sqlalchemy import Column, Integer, String, DateTime, create_engine
from sqlalchemy.orm import DeclarativeBase, Session

class Base(DeclarativeBase):
    pass

class Link(Base):
    __tablename__ = "links"
    id = Column(Integer, primary_key=True)
    code = Column(String(6), unique=True, index=True)
    original_url = Column(String, nullable=False)
    clicks = Column(Integer, default=0)
    created_at = Column(DateTime, server_default=func.now())
```

## 🏆 通关项目：Shortly — URL 短链服务

### API 设计

```
POST /api/shorten          → 创建短链接
GET  /api/links            → 列出所有短链（分页）
GET  /api/links/{code}     → 短链详情 + 点击统计
GET  /{code}               → 302 重定向到原始 URL
DELETE /api/links/{code}   → 删除短链
```

### 前端页面

- `/` — 输入长 URL → 生成短链 → 一键复制
- `/dashboard` — 短链列表 + 点击量排行

### 功能 → 技能映射

| 功能 | 练到的技能 |
|------|-----------|
| 短码生成（6位随机，不重复） | `secrets` 模块、唯一性检查 |
| URL 合法性校验 | `field_validator`、`HttpUrl` 类型 |
| 302 重定向 | `RedirectResponse`、HTTP 状态码 |
| 点击统计（时间、IP、UA） | SQLAlchemy 关联表 |
| 分页列表 | `Query` 参数、分页逻辑 |
| SQLite 持久化 | ORM Model、async session |
| 8 个 pytest 测试 | `httpx.AsyncClient` 测试 FastAPI |

## 🎮 章节小游戏：SQL 侦探 · 咖啡店失窃案

用 FastAPI + SQLAlchemy 构建一个侦探推理 API 游戏。数据库里藏着案件线索，玩家通过 API 查询破案。

### 剧情

```
☕ 咖啡店老板说：昨晚闭店后，保险柜里的 5000 元现金失踪了。
监控坏了，但刷卡系统有记录。有 6 名员工昨晚值班。
请查阅记录，找出嫌疑人。

可用的 API：
  GET /api/employees           → 查看所有员工信息
  GET /api/shifts?date=2025-03-22  → 查看当天排班
  GET /api/access_logs         → 查看门禁刷卡记录
  GET /api/transactions        → 查看当天交易流水
  POST /api/accuse             → 提交嫌疑人 {"employee_id": 3, "reason": "..."}
```

### 需要用到的知识点

| 功能 | 知识点 |
|------|--------|
| 数据库表设计 | SQLAlchemy Model（员工、排班、门禁、交易） |
| 关联查询 | `relationship()`、`join` |
| 初始化种子数据 | `seed.py` 脚本灌入线索数据 |
| 查询过滤 | `Query` 参数筛选 |
| 破案验证 | 判断玩家提交答案是否正确 |

### 最低要求

- [ ] 4 张数据表（employees、shifts、access_logs、transactions）
- [ ] 灌入的数据中暗藏 3 条关键线索
- [ ] 玩家通过 API 查询能拼出真相
- [ ] 提交正确嫌疑人 → 返回 "🎉 破案成功！"

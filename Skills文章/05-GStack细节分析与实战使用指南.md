# GStack 细节分析与实战使用指南

> 🕹️ 当你厌倦了让人工智能（Claude Code）当作“打字员”后，你会发现：如果想把一个创意真正变成一个交付的产品，你需要的不是一个懂所有代码的机器，而是一个 **开发团队**。这正是 Y Combinator 首席执行官 Garry Tan 提出 **GStack** 这一套极具实战精神的技能工作流的初衷。

---

### 背景引入：打破“大锅饭 prompt”的迷局

很多人使用 Claude Code 时习惯说：“用 React 和 Tailwind 写一个任务管理工具，别忘了存到 localStorage。”
它的结果往往是：生成一大堆拼凑的代码，遇到复杂逻辑就崩溃，最终的界面甚至难以跑通。

GStack 的哲学是：**化身团队、角色链式调用**。它把一个庞杂的开发任务硬拆成多个极具针对性的步骤（或称之为角色）。**不仅帮你写代码，更是帮你思考、推断、审查和测试。**

---

## 一、环境搭建与配置：让 GStack 接管 Claude Code

GStack 是运行在 Anthropic 官方 CLI 工具 `Claude Code` 之上的。你可以按照以下步骤完成安装：

1️⃣ **克隆技能包到本地**：
```bash
git clone https://github.com/garrytan/gstack.git ~/.claude/skills/gstack
cd ~/.claude/skills/gstack
./setup
```

2️⃣ **配置项目级别的 `CLAUDE.md`**：
在你需要开始编写的大项目根目录下创建（或追加修改）`CLAUDE.md`：
```markdown
# 工作环境要求
如果你需要浏览网页、自动化测试网页交互或者抓取 DOM，强烈要求你使用 GStack 提供的 `/browse` 指令，代替默认行为。
允许使用的系统可用技能包括：`/plan-ceo-review`, `/plan-eng-review`, `/review`, `/qa`, `/ship`。
```
这时候，Claude Code 就被你的项目“收编”了，成为一支拥有纪律的团队。

## 二、四大核心绝学的细节深挖

GStack 是如何让 Claude 从外行变成正规军的？我们来详细拆解下面这四大指令：

### 1. 战略与产品定位：`/plan-ceo-review`
▪️ **核心意图**：这可能是 GStack 最有意思的模块，与其说它是个程序员助手，不如说它是你的虚拟合伙人（CEO 视角）。
▪️ **解决痛点**：开发者经常陷入技术自嗨，给一个无关紧要的功能叠加无数花哨的动效。而 `/plan-ceo-review` 的作用是：当你提出一个功能创意时，它会拿着“MVP 清单”残忍地质问你。
▪️ **核心产出**：它会帮你评估当前的特性是否属于“核心痛点”，并强制削减、砍掉边缘功能，输出一个极其克制的需求列表。

### 2. 技术蓝图规划：`/plan-eng-review`
▪️ **核心意图**：产品需求落锤后，这是团队 Tech Lead（技术总监）登场的环节。
▪️ **解决痛点**：不经过设计就开始编码，必然导致数据状态错乱和代码难以扩展（也就是典型的“屎山代码”）。
▪️ **核心产出**：在写任何一行代码前，它会严格为你梳理状态管理架构、定义各个数据模块。例如它会指出“你在前端没做数据持久化处理”，并规划出后续必须的组件目录和类型定义。

### 3. 代码的试金石：`/review`
▪️ **核心意图**：这是一位严苛的代码审查员。
▪️ **解决痛点**：不要以为 AI 写的代码就没有逻辑漏洞。在通过 Claude 自动生成代码之后，直接跑 `/review`。
▪️ **核心产出**：它不再只关注有没有少了结束括号。它会进行**对抗性审查**：寻找潜在的内存溢出、死循环、异步锁定问题，或者未能考虑到错误边界的极端用例，并为你提供针对性的性能改进建议。

### 4. 端到端自动化防线：`/qa` (配合 `/browse`)
▪️ **核心意图**：打破了过去 AI 只能做单元测试的魔咒。
▪️ **解决痛点**：有时候前端代码没错，但是网页的输入框在某些特定的高度下点不到。
▪️ **核心产出**：由于 GStack 强制让 AI 调用了一套包含持久会话的无头浏览器（比如 Playwright 环境的 `/browse`），使用 `/qa` 时，AI 竟然能在真实的浏览器中渲染你的页面、填写表单、点击按钮，然后告诉你控制台抛出了哪些奇怪的报错，最终直接把带着截图的 Bug Issue 拍在你的脸上！

## 三、最佳实践：全链路“黑客马拉松”工作流

掌握完兵器，怎么使用它才能发挥最大威力？
Gary Tan 给出的标准答案是：**链式调用闭环**。

> 💡 **实战案例演绎（从零到一）**：
> 1. 你：打开终端唤出 Claude Code，输入 `我想重构我的仪表盘。`
> 2. 执行 **`/plan-ceo-review`** -> AI 会砍掉你乱加的动画，定下只保留数据和表格。
> 3. 拿到简要文档后，紧接着让它进行 **`/plan-eng-review`** -> AI 根据上一条输出规划这几个组件的数据流和 TypeScript 接口。
> 4. 让 Claude 按照清单**开始干活（写代码代码）**。
> 5. 提交前，执行 **`/review`** -> 找出一处组件异步渲染问题并让它主动修掉。
> 6. 执行 **`/qa`** -> 虚拟浏览器自动点进你的开发服务器做 UI 走查，确认页面没有崩溃。
> 7. 最后，运行 **`/ship`** -> 打包、走查代码覆盖度无误后自动在 Git 上创建 Pull Request。

整个流程环环相扣：产品 → 架构 → 研发 → 审稿 → 测试 → 交付。

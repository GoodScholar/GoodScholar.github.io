# 全局文章日期验证 Agent

## 需求背景

博客项目在日常文章编写过程中，偶尔出现以下问题：
- 文章日期被设定为**未来日期**，导致发布时时间不合理
- 多篇文章使用了**相同日期**，导致排序和展示异常

需要通过 AI Agent 全局规则自动拦截这两类问题。

## 功能需求

### 1. 日期不超过当前时间

AI 在创建或修改文章日期时，必须确保日期 ≤ 当前日期。

**检查范围**：
- YAML Frontmatter 的 `date` 字段
- 文章末尾签名行的 `更新日期：YYYY-MM-DD`

### 2. 日期不可重复

AI 在设定文章日期前，必须扫描项目中所有已有文章的日期，确保不重复。

**检查范围**：
- 全局所有 `.md` 文章文件

### 3. 两处日期保持一致

若同一篇文章同时存在 frontmatter `date` 和末尾签名行的更新日期，两者必须相同。

## 实现方式

通过在项目根目录创建 `AGENTS.md` 文件，定义全局 Agent 指令规则。AI 编辑器（如 Gemini、Claude Code、Cursor 等）会自动读取并遵循这些规则。

## 验收标准

- [x] 项目根目录存在 `AGENTS.md` 文件
- [x] 文件包含日期不超过当前时间的规则
- [x] 文件包含日期不可重复的规则
- [x] 文件包含两处日期保持一致的规则
